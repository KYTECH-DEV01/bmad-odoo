Pyramid Model — Foundation First

Architectural frame for the 2026-05-21 refactor · Initiative 2026-05-21__odoo-biz-refactor

Current reality: Odoo ERP is live · Domain workflows are evolving ad-hoc · AI capability is fragmented.

Goal: Consolidate the three into one model/architecture that supports big scale and growth hack.

Principle: Bottom-up — strengthen the foundation before AI takes off.

Layer 3 · Top

AI Capability

Current:
  • COO, SCM agents, BI map, skills standing apart
  • No compounding — each tool is a silo
  • Sits on noisy data — hard to trust outputs
Target:
  • Biz-grounded · compound · scale-ready
  • Prescriptive Radar guides employees in-the-moment
  • Agent platform consolidated around the single SoT
Layer 2 · Middle

Odoo ERP

Current:
  • Workflows written around screens, not the business
  • Bloated — duplicated artifacts (3 transport slips, 4 payment-request entry points)
  • Hard to onboard ordinary employees
Target:
  • 100% reflects business domain
  • Single source of truth for data + workflow
  • Easy onboard · everyday employee language
Layer 1 · Base

Business Domain

Current:
  • Workflow evolves ad-hoc
  • Business language not standardized
  • SOPs scattered, unversioned
Target:
  • Clear · biz-language · employee-first
  • SOPs versioned per domain
  • Service Lines + Journey + Actors + Flows + Branches (L1–L5)

Refactor order — Bottom-Up

  1. Strengthen Layer 1 — Business Domain Current · L1–L5 biz story

    P1 Domain Cartography + P2 Workflow Rewrite. Output: Domain Map, biz-language Glossary, biz-first SOPs per domain. This must finish before touching Odoo.

  2. Align Layer 2 — Odoo ERP

    P3 Gap Analysis (compare biz-first workflow vs current Odoo) + P4 Restructure Plan (rename fields/menus/screens to biz language, cut bloat, fill gaps). Output: Gap Matrix + Restructure roadmap. Manh approves each module C/U/D.

  3. Empower Layer 3 — AI Capability

    P5+ AI consolidation. Includes: Operations Radar (prescriptive UI inline in Odoo, blocking bad data), agent platform consolidation (COO + SCM + bots around the single SoT), automation. Only feasible once base & middle are solid — otherwise AI just amplifies noisy data.
